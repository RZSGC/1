# 校园在线投票系统课程展示版设计

## 背景

本项目是一个使用 JavaWeb + 数据库技术完成的校园在线投票系统，适用于校园活动、民意调查、班级评选和娱乐互动。系统已经具备 Servlet、JSP、JDBC、H2 数据库、`main()` 启动、防重复提交和 CSS 进度条结果展示能力。

本次设计目标是把项目整理成更适合课程展示和答辩的版本，重点强化管理员端和用户端的系统分工。

## 设计目标

- 管理员端像一个完整后台，而不是单个创建表单。
- 用户端保留清晰的投票大厅、投票详情和实时结果展示。
- 技术实现保持在 JavaWeb + 数据库范围内，不引入复杂框架。
- 防重复提交逻辑继续由后端事务和数据库联合唯一索引保障。
- CSS 进度条继续由后端计算的票数百分比动态驱动。

## 范围

### 本次包含

- 管理员端新增“已发布投票列表”。
- 管理员端新增统计概览：投票总数、累计票数、系统能力提示。
- 管理员端每条投票显示标题、说明、总票数和查看结果入口。
- `AdminServlet` 在 GET 请求中查询全部投票并传给 JSP。
- README 补充管理员系统页面设计说明。

### 本次不包含

- 管理员登录认证。
- 删除投票。
- 开启或关闭投票。
- 多角色权限系统。
- 前后端分离或 Spring 框架改造。

这些功能会增加数据库字段、权限判断和更多 Servlet，不适合作为本轮课程展示版的核心范围。

## 系统角色

### 管理员端

管理员负责发起和管理投票主题。管理员页面应包含两个主要区域：

- 创建投票：填写投票标题、说明和多个选项，提交后创建数据库记录。
- 已发布投票：查看所有投票主题、说明、累计票数，并进入结果页。

管理员端页面路径：

```text
/vote/admin
```

### 用户端

用户负责浏览投票、参与投票和查看实时民意结果。用户页面包括：

- 投票大厅：展示所有可参与投票。
- 投票详情：填写姓名或学号，选择选项并提交。
- 实时结果：展示票数、百分比和 CSS 进度条。

用户端页面路径：

```text
/vote/
/vote/poll?id=投票编号
```

## 架构设计

项目继续保持 MVC 风格：

- Model：`Poll`、`PollOption` 表示投票主题和选项。
- DAO：`PollDao` 负责数据库查询、创建投票、投票提交和结果统计。
- Controller：Servlet 负责请求分发和参数校验。
- View：JSP 负责页面渲染，CSS 负责视觉样式。

`AdminServlet` 是本次主要改动点：

- `doGet` 查询所有投票列表。
- 将投票列表、投票数量、累计票数放入 request。
- 转发到 `admin.jsp`。

`admin.jsp` 是本次主要页面改动点：

- 保留创建投票表单。
- 新增后台统计概览。
- 新增已发布投票列表。

## 数据流

### 创建投票

1. 管理员访问 `/admin`。
2. Servlet 查询已有投票并展示后台页面。
3. 管理员提交标题、说明和选项。
4. `AdminServlet#doPost` 校验标题和选项数量。
5. `PollDao#createPoll` 使用事务写入 `polls` 和 `poll_options`。
6. 创建成功后跳转到投票详情页。

### 用户投票

1. 用户从首页选择投票主题。
2. `PollServlet` 查询投票详情和选项统计。
3. 用户提交姓名或学号以及选项编号。
4. `VoteServlet` 生成 `voter_key`。
5. `PollDao#vote` 在事务中插入投票记录并更新选项票数。
6. 如果违反 `poll_id + voter_key` 唯一约束，返回重复投票提示。

## 防重复提交设计

数据库表 `votes` 使用联合唯一索引：

```sql
CONSTRAINT uk_vote_once UNIQUE (poll_id, voter_key)
```

后端使用：

```text
姓名或学号 + IP
```

生成 `voter_key`。同一用户对同一投票重复提交时，数据库会拒绝插入，DAO 捕获约束异常并返回重复提交结果。这样既有后端逻辑控制，也有数据库层面的强约束。

## 实时结果设计

后端在查询投票详情时计算：

```text
选项百分比 = 当前选项票数 / 总票数 * 100
```

JSP 使用百分比设置 CSS 进度条宽度：

```jsp
style="width: <%= percent %>%"
```

这样页面展示与数据库票数保持同步，符合课程要求中的“前端 CSS 进度条与后端票数比例数据动态绑定”。

### 管理员后台

- 创建投票区：主题、说明、选项输入框。

### 用户端

页面结构：

- 用户首页：投票大厅，展示投票列表。
- 投票详情：投票表单和实时结果。

- 用户端更轻量、清晰，强调参与投票。
- 结果区突出票数、百分比和进度条。

## 错误处理

- 创建投票时标题为空或选项少于两个，返回管理员页面并显示错误提示。
- 访问不存在的投票编号，返回 404。
- 用户未填写姓名或未选择选项，返回提示。
- 重复投票时，返回“不能重复提交”的业务提示。

## 测试方案

手动测试重点：

1. 使用 `Main.java` 启动系统。
2. 访问 `/vote/admin` 创建一个投票。
3. 确认管理员端列表出现新投票。
4. 点击查看结果进入详情页。
5. 使用不同姓名提交投票，确认票数增加。
6. 使用相同姓名和同一浏览器地址重复提交，确认系统提示不能重复提交。
7. 确认实时结果百分比和 CSS 进度条宽度同步变化。

编译测试：

```bash
mvn -q -DskipTests compile
```

运行测试：

```bash
mvn compile exec:java
```

访问：

```text
http://localhost:8080/vote/
```
