# 校园在线投票系统

这是一个使用 JavaWeb + 数据库技术完成的在线投票平台，适用于校园活动、民意调查、班级评选和娱乐互动场景。系统包含登录注册、角色区分、管理员后台和用户投票端：管理员负责创建投票主题和查看数据，用户登录后参与投票并查看票数、百分比和 CSS 进度条统计结果。

## 一、项目设计目标

本项目围绕“投票发布、用户参与、结果统计、防止重复提交”四个核心流程设计。管理员可以快速发布投票，用户根据投票主题进行选择，后端负责保存数据、计算票数比例，并通过 JSP 页面动态展示统计结果。

系统重点体现两个 JavaWeb + 数据库技术点：

- 防止重复提交：使用 `poll_id + voter_key` 联合唯一索引，并在后端根据投票结果给出提示。
- 动态结果展示：后端计算每个选项的票数和百分比，JSP 将百分比绑定到 CSS 进度条宽度。
- 角色权限区分：账号注册时选择普通用户或管理员，登录后进入不同系统页面。

## 二、系统角色设计

### 管理员端

管理员端主要用于投票管理和活动发布。

- 使用管理员账号登录后台
- 创建投票主题
- 填写活动说明或投票规则
- 设置多个投票选项
- 提交后自动进入投票详情页
- 查看后台统计概览，包括已发布投票数量和累计票数
- 查看已发布投票列表，并进入指定投票的结果页
- 便于课堂演示、活动现场展示和结果查看

管理员页面：

```text
http://localhost:8080/vote/admin
```

管理员端通过独立后台地址访问，不在用户投票流程中暴露后台操作入口。

### 用户端

用户端主要用于浏览投票、参与投票和查看实时结果。

- 使用普通用户账号登录系统
- 查看所有可参与投票
- 进入指定投票详情页
- 选择一个选项并提交
- 查看实时票数、比例和进度条
- 重复投票时系统提示不能重复提交

用户首页：

```text
http://localhost:8080/vote/
```

用户端只展示投票大厅、投票详情和实时结果，不提供投票创建或后台管理操作。

## 三、页面设计

### 用户首页

登录首页提供账号登录和账号注册。用户注册时选择普通用户或管理员角色，登录后系统根据角色自动跳转：普通用户进入投票大厅，管理员进入后台管理页面。

用户首页定位为“校园投票大厅”，只保留用户需要的浏览和参与入口。页面展示当前所有投票主题，每个投票卡片显示投票编号、标题、说明和总票数，用户点击“参与投票”进入详情页。用户页面不出现创建投票、后台管理等管理员功能。

### 管理员后台

管理员后台定位为“课程展示版投票发布后台”，与用户端视觉和功能分开。页面包含统计概览、投票创建表单和已发布投票列表。统计概览展示已发布投票数量、累计票数和系统能力；创建区负责填写主题、说明和选项；管理区展示所有投票主题、说明、票数和查看结果入口。

### 投票详情页

投票详情页同时包含用户投票区和实时结果区。用户提交后，系统跳回详情页并展示提示信息。实时结果区根据数据库中的票数计算百分比，并用 CSS 进度条展示民意分布。

## 五、数据库设计

系统包含四张核心表：

- `users`：账号表，保存登录账号、密码摘要、昵称和角色。
- `polls`：投票主题表，保存标题、说明、创建时间。
- `poll_options`：投票选项表，保存选项文本和票数。
- `votes`：投票记录表，保存用户投票记录。

防止重复提交的关键约束：

```sql
CONSTRAINT uk_vote_once UNIQUE (poll_id, voter_key)
```

其中 `voter_key` 由登录用户账号编号生成。这样可以在后端校验的基础上，再通过数据库唯一约束防止同一用户对同一投票重复提交。

## 六、运行方式

方式一：使用 `main()` 启动。

直接运行：

```text
src/main/java/com/campus/vote/Main.java
```

或使用 Maven：

```bash
mvn compile exec:java
```

方式二：使用 Jetty Maven 插件启动。

```bash
mvn jetty:run
```

启动后访问：

```text
http://localhost:8080/
http://localhost:8080/vote/login
http://localhost:8080/vote/
```

访问根路径 `http://localhost:8080/` 会自动跳转到登录首页 `/vote/login`。

## 八、项目特色

- 管理员端和用户端页面分工明确。
- 管理员端包含统计概览、创建表单和已发布投票列表。
- 使用 Servlet + JSP 完成请求处理和页面渲染。
- 使用 JDBC 操作数据库，符合 JavaWeb 课程项目要求。
- 使用 SQLite 数据库，首次启动自动创建数据文件，便于使用 Navicat 查看。
- 使用联合唯一索引实现可靠的防重复投票控制。
- 使用后端统计数据动态驱动前端 CSS 进度条。
